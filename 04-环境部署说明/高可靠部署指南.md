# 物业管理系统部署指南（非 Docker）

## 一、环境要求

| 组件 | 版本要求 | 用途 |
|------|----------|------|
| JDK | 17+ | 后端运行 |
| Maven | 3.6+ | 后端构建 |
| Node.js | 18+ | 前端构建 |
| MySQL | 8.0+ | 数据库 |
| Nginx | 1.20+ | 反向代理 + 负载均衡 |

## 二、部署架构

```
用户请求
   │
   ▼
┌─────────┐
│  Nginx  │  :80  负载均衡 + 静态资源
└────┬────┘
     │
     ├──────────┬──────────┐
     ▼          ▼          ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│Backend-1│ │Backend-2│ │Backend-3│
│  :2010  │ │  :2011  │ │  :2012  │
└────┬────┘ └────┬────┘ └────┬────┘
     └───────────┴───────────┘
                 │
                 ▼
           ┌───────────┐
           │   MySQL   │  :3306
           └───────────┘
```

## 三、部署步骤

### 步骤 1：初始化数据库

```bash
# 登录 MySQL
mysql -u root -p

# 执行建库脚本
source 02-SQL完整脚本/01_create_database.sql

# 执行表结构
source 02-SQL完整脚本/02_schema.sql

# 导入测试数据
source 02-SQL完整脚本/03_mock_data.sql

# 导入操作日志表
source 02-SQL完整脚本/04_operation_log.sql
```

验证：
```bash
mysql -u root -p -e "USE rb_pms; SHOW TABLES;"
# 应看到：sys_user, prop_building, prop_unit, owner, repair_request 等表
```

### 步骤 2：构建后端

```bash
cd 01-项目源码/backend

# 清理并打包
mvn clean package -DskipTests

# 验证 JAR 生成
ls target/property-management-system-1.0.0.jar
```

### 步骤 3：构建前端

```bash
# 构建管理端
cd 01-项目源码/frontend-admin
npm install   # 首次需要
npm run build

# 构建业主端
cd ../frontend-client
npm install
npm run build
```

验证：
```bash
ls frontend-admin/dist/index.html
ls frontend-client/dist/index.html
```

### 步骤 4：启动后端多实例

**方式 A：PowerShell 脚本（推荐）**

```bash
# 启动单个实例（端口 2010）
.\scripts\start-backend.ps1 -Port 2010

# 启动 3 个实例
.\scripts\start-backend.ps1 -Port all
```

**方式 B：手动启动**

```bash
# 实例 1
SERVER_PORT=2010 java -Xms512m -Xmx1024m -jar 01-项目源码/backend/target/property-management-system-1.0.0.jar

# 实例 2（新终端）
SERVER_PORT=2011 java -Xms512m -Xmx1024m -jar 01-项目源码/backend/target/property-management-system-1.0.0.jar

# 实例 3（新终端）
SERVER_PORT=2012 java -Xms512m -Xmx1024m -jar 01-项目源码/backend/target/property-management-system-1.0.0.jar
```

验证：
```bash
curl http://localhost:2010/actuator/health
# 返回 {"status":"UP","components":{"db":{"status":"UP"},...}}
```

### 步骤 5：安装配置 Nginx

```bash
# 下载 Nginx（Windows）
# 访问 http://nginx.org/en/download.html 下载 stable 版本
# 解压到 C:\nginx
```

编辑 `C:\nginx\conf\nginx.conf`，替换为以下内容：

```nginx
worker_processes auto;
error_log logs/error.log warn;
pid logs/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent"';

    access_log logs/access.log main;

    sendfile        on;
    tcp_nopush      on;
    tcp_nodelay     on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript
               text/xml application/xml text/javascript image/svg+xml;

    # 后端负载均衡
    upstream backend {
        least_conn;
        server 127.0.0.1:2010 max_fails=3 fail_timeout=30s;
        server 127.0.0.1:2011 max_fails=3 fail_timeout=30s;
        server 127.0.0.1:2012 max_fails=3 fail_timeout=30s;
    }

    server {
        listen 80;
        server_name _;

        # 管理端前端
        location /admin/ {
            alias /你的路径/frontend-admin/dist/;
            try_files $uri $uri/ /admin/index.html;

            location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
                expires 30d;
                add_header Cache-Control "public, immutable";
            }
        }

        # 业主端前端
        location /client/ {
            alias /你的路径/frontend-client/dist/;
            try_files $uri $uri/ /client/index.html;

            location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
                expires 30d;
                add_header Cache-Control "public, immutable";
            }
        }

        # API 反向代理（自动故障转移）
        location /api/ {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;

            proxy_connect_timeout 5s;
            proxy_read_timeout 30s;
            proxy_send_timeout 10s;

            proxy_next_upstream error timeout http_502 http_503 http_504;
            proxy_next_upstream_tries 3;
            proxy_next_upstream_timeout 10s;
        }

        # 文件上传代理
        location /resources/uploads/ {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        # 默认跳转管理端
        location = / {
            return 302 /admin/;
        }
    }
}
```

> **注意**：将 `alias` 后的路径替换为前端 dist 目录的实际绝对路径，使用正斜杠 `/`。

启动 Nginx：
```bash
cd C:\nginx
nginx.exe          # 启动
nginx.exe -s stop  # 停止
nginx.exe -s reload # 重载配置
```

验证：
```bash
curl http://localhost/admin/
curl http://localhost/client/
curl http://localhost/api/dashboard/stats
```

## 五、日常运维

### 启停服务

```bash
# 启动所有后端实例
.\scripts\start-backend.ps1 -Port all

# 停止所有实例（Windows）
taskkill /F /IM java.exe

# 重启 Nginx
cd C:\nginx
nginx.exe -s reload
```

### 查看日志

```bash
# Nginx 日志
type C:\nginx\logs\access.log
type C:\nginx\logs\error.log

# 后端日志（控制台输出）
# 生产环境可配置 logback-spring.xml 输出到文件
```

### 数据库备份

```bash
# 手动备份
mysqldump -u root -p rb_pms | gzip > backup_$(date +%Y%m%d).sql.gz

# 定时备份（Linux/Mac）
crontab -e
# 添加：0 3 * * * /path/to/scripts/backup-db.sh /backup
```

### 健康检查

```bash
# 检查后端是否存活
curl http://localhost:2010/actuator/health

# 批量检查所有实例
curl http://localhost:2010/actuator/health | grep '"status":"UP"'
curl http://localhost:2011/actuator/health | grep '"status":"UP"'
curl http://localhost:2012/actuator/health | grep '"status":"UP"'
```

## 六、故障排查

| 问题 | 排查步骤 |
|------|----------|
| 后端启动失败 | 检查 MySQL 是否运行：`mysql -u root -p -e "SELECT 1"` |
| Nginx 502 | 后端未启动或端口冲突：`netstat -ano | findstr 2010` |
| 页面空白 | 检查 `alias` 路径是否正确，dist 目录是否存在 |
| API 超时 | 检查后端日志，可能是数据库慢查询 |
| CORS 错误 | 确认 `application.yml` 中 `cors-origins` 包含 Nginx 地址 |
