# 物业管理系统 — 高可用可靠性设计方案

## 一、现状分析

### 当前架构
- 后端: Spring Boot 3.2.5 单实例 (port 2010)
- 前端: Vue 3 × 2 (admin:4010, client:3010) — Vite dev server
- 数据库: MySQL 8 单实例 (root/123456)
- 无 Docker/容器化, 无负载均衡, 无监控

### 可靠性风险点
| 风险 | 影响 | 当前状态 |
|------|------|----------|
| 后端单实例挂掉 | 整个系统不可用 | 无重启/冗余 |
| MySQL 单实例故障 | 数据丢失+服务中断 | 无主从/备份 |
| 无健康检查 | 故障无法被发现 | 无 |
| 无日志聚合 | 排查问题困难 | 仅控制台日志 |
| 前端无生产构建 | 开发服务器不适合生产 | Vite dev only |
| 文件存储在本地 | 服务器重启文件可能丢失 | 本地磁盘 |
| JWT 无黑名单 | Token 无法主动失效 | 无 |
| 无 API 限流 | 可能被恶意请求打垮 | 无 |

### 模块 A: 后端多实例部署 + 健康检查

**目标**: 后端服务多实例运行，单实例故障不影响整体服务

#### A1. 添加 Spring Boot Actuator 健康检查端点

**文件**: `pom.xml` — 添加依赖
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

**文件**: `application.yml` — 添加配置
```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics
  endpoint:
    health:
      show-details: always
  health:
    db:
      enabled: true
    diskspace:
      enabled: true
```

暴露端点:
- `GET /actuator/health` — 应用健康状态 (含数据库连接状态)
- `GET /actuator/metrics` — JVM/HTTP 指标

**文件**: `WebConfig.java` — CORS 添加 actuator 端点, 排除鉴权
**文件**: `JwtAuthenticationFilter.java` — EXCLUDE_PATHS 添加 `/actuator`

#### A2. 多实例启动脚本

**新建文件**: `scripts/start-backend.sh` (Linux) / `scripts/start-backend.bat` (Windows)
- 支持通过环境变量指定端口: `SERVER_PORT=2010 java -jar backend.jar`
- 支持启动多个实例 (2010, 2011, 2012)
- 内置 JVM 参数优化: `-Xms512m -Xmx1024m -XX:+UseG1GC`

#### A3. 数据库连接池优化

**文件**: `application.yml` — 添加 HikariCP 配置
```yaml
spring:
  datasource:
    hikari:
      minimum-idle: 5
      maximum-pool-size: 20
      idle-timeout: 300000
      max-lifetime: 1200000
      connection-timeout: 30000
      leak-detection-threshold: 60000
```

### 模块 B: 数据库主从复制 + 自动切换

**目标**: MySQL 主从复制，主库故障时自动切换到从库

#### B1. MySQL 主从复制配置

**新建文件**: `03-deploy/mysql/master.cnf` — 主库配置
```ini
[mysqld]
server-id=1
log-bin=mysql-bin
binlog-format=ROW
gtid-mode=ON
enforce-gtid-consistency=ON
```

**新建文件**: `03-deploy/mysql/slave.cnf` — 从库配置
```ini
[mysqld]
server-id=2
relay-log=relay-bin
read-only=ON
gtid-mode=ON
enforce-gtid-consistency=ON
```

**新建文件**: `03-deploy/mysql/init-replication.sql` — 复制用户和授权
```sql
CREATE USER 'repl_user'@'%' IDENTIFIED BY 'repl_password';
GRANT REPLICATION SLAVE ON *.* TO 'repl_user'@'%';
CHANGE MASTER TO ...;
START SLAVE;
```

#### B2. Spring Boot 多数据源 + 自动切换

**文件**: `pom.xml` — 添加依赖
```xml
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>dynamic-datasource-spring-boot3-starter</artifactId>
    <version>4.3.1</version>
</dependency>
```

**文件**: `application.yml` — 多数据源配置
```yaml
spring:
  datasource:
    dynamic:
      primary: master
      strict: false
      datasource:
        master:
          url: jdbc:mysql://master-host:3306/rb_pms?...
          username: root
          password: 123456
        slave:
          url: jdbc:mysql://slave-host:3306/rb_pms?...
          username: root
          password: 123456
```

**新建文件**: `config/DatasourceHealthIndicator.java` — 自定义健康检查，检测主库连通性
**新建文件**: `config/FailoverService.java` — 主库不可达时自动切换到从库

#### B3. 数据库备份策略

**新建文件**: `scripts/backup-db.sh` — 定时备份脚本
```bash
# 每天凌晨3点全量备份
mysqldump -u root -p rb_pms | gzip > /backup/rb_pms_$(date +%Y%m%d).sql.gz
# 保留最近30天
find /backup -name "*.sql.gz" -mtime +30 -delete
```

**新建文件**: `scripts/crontab.txt` — cron 任务配置

### 模块 C: Nginx 负载均衡 + 高可用

**目标**: Nginx 作为反向代理，实现负载均衡和静态资源服务

#### C1. Nginx 配置

**新建文件**: `03-deploy/nginx/nginx.conf`
```nginx
upstream backend {
    least_conn;
    server localhost:2010 max_fails=3 fail_timeout=30s;
    server localhost:2011 max_fails=3 fail_timeout=30s;
    server localhost:2012 max_fails=3 fail_timeout=30s;
}

server {
    listen 80;
    server_name your-domain.com;

    # 前端静态资源
    location /admin/ {
        alias /opt/frontend-admin/dist/;
        try_files $uri $uri/ /admin/index.html;
    }
    location /client/ {
        alias /opt/frontend-client/dist/;
        try_files $uri $uri/ /client/index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_connect_timeout 5s;
        proxy_read_timeout 30s;
        proxy_send_timeout 10s;
        proxy_next_upstream error timeout http_502 http_503;
    }

    # 上传文件代理
    location /resources/uploads/ {
        proxy_pass http://backend;
    }
}
```

关键配置:
- `least_conn` — 最少连接负载均衡
- `max_fails=3 fail_timeout=30s` — 3次失败后标记为不可用，30秒后重试
- `proxy_next_upstream` — 自动重试下一个后端实例

#### D2. 前端请求重试机制

**文件**: `frontend-admin/src/api/request.js` — 添加 axios 重试
```js
import axios from 'axios'

const request = axios.create({ baseURL: '/api', timeout: 10000 })

// 请求重试：网络错误自动重试2次
request.interceptors.response.use(
  (res) => { /* 保持现有逻辑 */ },
  (err) => {
    const config = err.config
    if (!config || config._retry) return Promise.reject(err)

    // 网络错误或5xx错误自动重试
    if (!err.response || err.response.status >= 500) {
      config._retry = (config._retry || 0) + 1
      if (config._retry <= 2) {
        return new Promise(resolve => {
          setTimeout(() => resolve(request(config)), 1000 * config._retry)
        })
      }
    }

    if (err.response?.status === 401) {
      localStorage.removeItem('user')
      localStorage.removeItem('admin_menus')
      router.push('/login')
    }
    return Promise.reject(err)
  }
)
```

**文件**: `frontend-client/src/api/request.js` — 同上

**新建文件**: `03-deploy/docker/Dockerfile前端` (admin + client)

#### F2. Docker Compose 编排

**新建文件**: `03-deploy/docker-compose.yml`
```yaml
version: '3.8'
services:
  mysql-master:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: rb_pms
    volumes:
      - mysql-master-data:/var/lib/mysql
      - ./mysql/master.cnf:/etc/mysql/conf.d/master.cnf
    ports:
      - "3306:3306"

  mysql-slave:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: 123456
    volumes:
      - mysql-slave-data:/var/lib/mysql
      - ./mysql/slave.cnf:/etc/mysql/conf.d/slave.cnf
    ports:
      - "3307:3306"
    depends_on:
      - mysql-master

  backend-1:
    build:
      context: ../../01-项目源码/backend
      dockerfile: ../../03-deploy/docker/Dockerfile.backend
    environment:
      SERVER_PORT: 2010
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql-master:3306/rb_pms?...
    depends_on:
      - mysql-master

  backend-2:
    build:
      context: ../../01-项目源码/backend
      dockerfile: ../../03-deploy/docker/Dockerfile.backend
    environment:
      SERVER_PORT: 2011
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql-master:3306/rb_pms?...
    depends_on:
      - mysql-master

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
      - ../../01-项目源码/frontend-admin/dist:/opt/frontend-admin/dist
      - ../../01-项目源码/frontend-client/dist:/opt/frontend-client/dist
    depends_on:
      - backend-1
      - backend-2

volumes:
  mysql-master-data:
  mysql-slave-data:
```
